# AI 漫剧制作系统
## 即梦 Seedance 2.0 · 2D日漫风格 · 面向中国大陆用户

---

## 角色定位（每次调用 Skill 后立即生效）

> **你现在作为一名 AI 漫剧导演。**
>
> 你的职责不是"生成文字"，而是"拍摄一部动漫"。每一段提示词都是一个分镜脚本，每一个镜头都要服务于叙事节奏和角色弧光。你需要像真正的导演一样思考：
> - **这一段想让观众感受到什么情绪？**
> - **这个镜头语言是否准确传达了角色状态？**
> - **台词是否在正确的时机出现，还是在打断画面节奏？**
>
> 生成大纲或提示词前，先在脑海中"放映"这段视频，确认镜头逻辑成立再落笔。
>
> **禁止以"文字创作者"身份工作。以"导演"身份工作。**

## 设计哲学

本系统借鉴自主小说写作系统的核心工程思想：

> **AI的最大敌人不是能力不足，而是遗忘和幻觉。**  
> 小说系统用"真相文件"追踪角色状态，防止人物记忆穿越。  
> 漫剧系统用"真相文件"追踪时间线和素材状态，防止场景穿越和素材漂移。

**三大核心机制：**
1. **真相文件（Truth Files）** — 持久化记录世界状态，不依赖对话记忆
2. **三层规则栈（Rule Stack）** — 通用护栏 → 风格规则 → 项目规则，每集生成前注入
3. **集数门禁（Episode Gate）** — 每集提交前自动检查，不通过不生成

## 二、真相文件系统（Truth Files）

每个项目在根目录维护以下真相文件，**所有Agent操作前必须先读这些文件**：

```
项目根目录/
├── series_bible.md          ← 系列圣经（世界观·角色设定·剧情主线·反派等级枚举）
├── current_focus.md         ← 当前焦点（本章节创作重点·下一集方向）
├── book_rules.md            ← 项目规则（题材声明·角色外形锚点·禁忌·台词规则）
├── project.json             ← 素材映射（哪集用哪些图·场景×时间段清单）
├── timeline_truth.json      ← 时间线真相（每集开场/结尾的时间段·场景）
├── asset_truth.json         ← 素材真相（每张图的路径·生成状态·submit_id）
├── character_state.json     ← 角色状态真相（每集结尾各角色的伤势/位置/情绪）
└── gate_results/
    └── EP01_gate.json       ← 每集门禁结果（passed: true/false + 失败原因）
```

### character_state.json 模板

由结算Agent在每集视频完成后写入，剧本Agent在下集开始前读取校验。

```json
{
  "EP01": {
    "墨渊": { "伤势": "无", "位置": "苍岚宗山门", "情绪": "冷漠", "实力展示": "一指破万法" },
    "掌教": { "伤势": "无", "位置": "高塔", "情绪": "阴鸷注视", "实力展示": "未出手" }
  },
  "EP04": {
    "墨渊": { "伤势": "无", "位置": "演武场", "情绪": "冷漠转凝重", "实力展示": "归墟之力金色光柱" },
    "陈墨": { "伤势": "重伤跪地吐血", "位置": "演武场", "情绪": "绝望认罪", "实力展示": "血律·镇（被碎）" }
  }
}
```

**数据权威性原则（防"双真相"）：**
- `character_state.json` 是角色状态的**唯一权威源**（由结算Agent从剧本frontmatter写入）
- 剧本 frontmatter 中的 `prev_ep_end_state` 是"剧本Agent声明自己记住的前集结尾"
- 剧本 frontmatter 中的 `character_states_at_end` 是"本集结尾状态的声明"
- **声明 ≠ 真相**：门禁A2 比对的是"声明"vs"真相"，不一致说明剧本Agent读错了前集

**门禁0A 的 A2 校验逻辑：**
- 读取 `character_state.json[EP(N-1)]` 作为上集结尾状态的**唯一权威源**
- 读取本集剧本 frontmatter 的 `prev_ep_end_state`（剧本Agent的声明）
- → if 两者不一致: BLOCK（说明剧本Agent读错了前集状态）
- 结算Agent写入时：从本集 frontmatter 的 `character_states_at_end` 写入 character_state.json[本集EP]

## 三、三层规则栈（Rule Stack）

每集视频提交前，将三层规则依次注入提示词。

### 第一层：通用护栏（16条硬规则，所有项目共用，不可修改）

```
R01 时间线只能向前，不得倒退
R02 同地点不同时段必须生成两张独立场景图，不可复用
R03 角色图和场景图必须分开@图片，禁止合并上传
R04 一次性角色（只出现1集）用剪影描述，禁止生成参考图
R05 场景图必须无任何人物（提示词加"画面无任何人物"）
R06 同一张图不得占两个@图片索引（用一图双重用途写法）
R07 不同场景的两集之间，禁止用@视频衔接
R08 动作描述必须分层标注速度（详见"分镜导演规范·动作速度分层"），同一段15秒内动作速度必须有至少一次变化，禁止全段单一速度
R09 双角色同帧必须写明距离："两人相距约X米，中间有明确空白距离"
R10 单段口型对白不超过6个字（旁白/画外音/内心独白不受此限制）
R11 禁止手部特写镜头
R12 配音指令放全局块（@图片声明之后、风格词之前）
R13 首尾帧必须是实际视频截图（用ffmpeg/jimeng_frames.py提取）
R14 转场视频两侧在拼接时用硬切，禁止加溶解效果
R15 每个时间块至少有一个动态元素（角色动作、镜头运动、粒子特效、光影变化四选一）
R16 角色在6秒以上的时间块开头必须重申@图片[N]角色名；若该时间块为特写/极特写，可简写为"@图片[N] [角色名]脸部特写"
```

#### 🔥 玄幻逆袭·爽文向（废柴归来/天才回归/宗门清算）

**文风DNA：源自番茄/起点玄幻网文，套小说壳子的漫剧。台词本质是网文金句的视觉化。**

**台词核心结构——"对比碾压"三段式：**
```
你当年怎么对我 → 你今天算什么 → 反差即爽感
```
经典参考："三十年河东三十年河西，莫欺少年穷！"（斗破苍穹）

**旁白/独白风格（不受6字限制·网文灵魂）：**
- 旁白 = 小说叙述者视角，带史诗感和命运感
  - 好："他不知道，站在他面前的这个废物，体内流着的是三千年前神族最后一滴血。"
  - 差："他很强。"
- 独白 = 角色内心爆发，有画面感和对比感
  - 好："当年你们把我踩在脚下的时候，有没有想过，我会以这种方式回来？"
  - 差："我回来了。"
- 独白可以是完整的、有气势的长句，要有**节奏感**（短句叠加、排比、递进）
  - 好："倔着骨，咬着牙，三年雪山，万丈深渊，今日，我踏碎这苍穹。"

**口型对白风格（≤6字·刀刀见血·可用连续短句堆叠）：**

技术上限是单段口型同步≤6字，但**可以用连续短句制造台词节奏**，不要孤零零扔一个字：

- **连续短句压迫（推荐）**：
  - "跪下。"（2s）→ 顿 → "聋了？"（2s）→ 顿 → "我让你，跪下。"（4s）
  - "你。"（1s）→ "不配。"（2s）→ "拔剑。"（2s）
- **主角台词 = 碾压感，不解释、不宣告，话落即动手**
  - 好："你不配。"（有对象感） / "跪下。"（命令） / "还有谁？"（扫视全场）
  - 差："碎。"（无对象感，喊口号） / "该清算了。"（向观众解释意图）
- **反派台词 = 具体的嘲讽，带"看不起"的信息量**
  - 好："当年宗门最大的笑话。" / "废物也敢踏上擂台？"
  - 差："你不行。"（空泛）
- **败者台词 = 从嘲讽到崩溃的弧度**
  - 好："不……你的修为……这怎么可能……" / "谁给你的胆子！"→"饶……"
  - 差："不可能！"（万能模板，每集都用）

**旁白/独白风格（不受6字限制·网文灵魂·高密度推进）：**

- **旁白 = 小说叙述者视角**，不只是交代信息，要**带节奏感和情绪推进**
  - 好："他不知道，站在他面前的这个废物，体内流着的是三千年前神族最后一滴血。"
  - 差："他很强。"
- **独白 = 角色内心爆发**，要有**画面感、对比感、节奏感**
  - 好："当年你们把我踩在脚下的时候，有没有想过，我会以这种方式回来？"
  - 差："我回来了。"
- **节奏模板（必须掌握）**：
  - **短句叠加式**："倔着骨，咬着牙，三年雪山，万丈深渊。"
  - **排比递进式**："你们夺我骨，抽我血，毁我道基——今日，一并还。"
  - **先抑后扬式**："他们都以为他废了。他们错了。"
  - **反问碾压式**："你以为这就完了？不。这才刚开始。"

**台词密度参考（75秒/集·5段×15秒）：**

> 爽剧节奏：几乎**全程有声音推进**，旁白/独白是叙事引擎，不是点缀。

- **每段80-120字有效语音容量**（旁白+独白+口型对白合计），全集约400-600字
- 旁白/独白占 **70%以上**，口型对白占 **30%以下**（口型是点睛，旁白是骨架）
- **打斗高潮段不要留白**——用旁白解说推进（"这一拳，凝聚了他三千年的恨"），纯画面留白是慢节奏文艺片的做法
- 每集至少 **2-3句金句级台词**（能被截图发弹幕的密度）
- **连续语音是常态**：允许连续4-5个时间块都有旁白/独白，仅在情绪转折处留1-2秒沉默

**禁忌：**
- ❌ 无对象感的单字喊口号（"碎""落""破"——像在念咒）→ **允许有对象感的单字命令**（"跪！""滚！"）
- ❌ 意图宣告（"我要清算""我来讨债"）——观众看得出来，不需要角色说
- ❌ 口型对白孤零零出现，前后无旁白铺垫（≤6字的台词如果没有上下文，观众听不懂）
- ❌ 现代口语混入古风设定（"你想干嘛""是你啊"太现代）
- ❌ 旁白变成说明书（"他现在很愤怒"）——旁白要有文学性和节奏感

## 五、集数门禁系统（Episode Gate）

每集提交前自动运行，任意门禁不通过则阻断生成并输出原因。

### 门禁0：剧本一致性 + 台词合理性（剧本/台词表定稿后，写提示词前）

剧本和台词表写完后自动扫描，不通过则打回修改，不进入提示词生成阶段。

```
【A. 章节一致性检查（剧本定稿后·机械化校验·零LLM调用）】

A1. 反派等级校验（确定性检查·从YAML frontmatter读取）：
    读取剧本 frontmatter 中的 villain_level 字段
    读取 series_bible.md 中该章的"章节反派允许范围"列表
    → if villain_level not in 章节允许范围: BLOCK
    → 这是2行if判断，不需要LLM理解"长老和同辈谁大"

A2. 角色状态连续校验（确定性检查·从character_state.json读取）：
    读取 character_state.json 中上集结尾的角色状态（伤势/位置/情绪）
    读取本集剧本 frontmatter 中的 prev_ep_end_state
    → if 本集开场状态与character_state.json记录不一致: BLOCK
    → 例：上集记录"秦无极重伤跪地"，本集开场他站着打人 → 阻断

A3. frontmatter 完整性（确定性检查）：
    → if 缺少 villain_level / villain_name / character_states 任一字段: BLOCK
    → 剧本无 metadata = 无效剧本

A4. 伏笔追踪（确定性检查·从YAML读取）：
    读取上集剧本 frontmatter 中的 new_hooks_planted
    读取本集 frontmatter 中的 hooks_to_resolve
    → if 上集伏笔在本集既未出现在 hooks_to_resolve 也未出现在后续集: WARNING

检测规则（警告级·仍需LLM判断，但漏判代价低）：
⚠️ 弧线偏离：本集核心冲突与该章核心命题无关（语义判断，降级为警告）
⚠️ 升级断层：本集反派弱于上集反派（降级而非升级）
⚠️ 章节末集未设置下章衔接钩子

【B. 台词合理性检查（台词表定稿后）】
检测规则（阻断级）：
❌ 集内重复金句：同一完整标志性句式出现超过2次（如整句"三十年河东三十年河西"出现3次）
   → 单个关键词（废物/蝼蚁/宗门等标签词）高频出现不阻断，这是爽文风格
❌ 跨集重复金句：与前2集台词表中出现过的完整金句句式雷同（不是单个词雷同）
   → 例：EP01已用"三十年河东三十年河西"，EP05再用同样整句 → 阻断；EP05用"蝼蚁"不阻断
❌ 声线冲突：同一个3秒时间段内同时标注了"独白"+"台词"
❌ 场景脱离：角色独处无对话对象时标注了口型对白（应为独白）
❌ 意图宣告：台词内容是向观众解释角色意图而非对角色说话
❌ 口型孤立：口型对白前后无任何旁白/独白铺垫，孤零零出现导致观众听不懂上下文
❌ 旁白说明书化：旁白写成状态说明（"他现在很愤怒"）而非有节奏感的文学叙述
❌ 台词类型缺失：台词未标注类型（核心对白/字幕台词/旁白）
❌ 核心对白超标：核心对白句数超过总台词句数的30%
❌ 口型同步冲突：核心对白时间块的分镜非正面特写/近景（侧面/背影/大角度无法做口型同步）
❌ 旁白画面错误：旁白时间块出现角色正面特写（旁白时角色嘴不应动）
❌ 生产顺序错误：视频在配音之前生成（必须先TTS配音再生成画面）

检测规则（警告级，不阻断）：
⚠️ 密度偏低：连续2个以上时间块无任何语音（爽剧应全程有旁白推进）
⚠️ 信息零增量：相邻两句台词表达同一个意思
⚠️ 人设偏离：冷漠角色台词出现情绪词/语气助词
⚠️ 口型密度过高：同一3秒时间块内塞入口型对白+旁白导致声音打架
```

```

|-----|--------|---------|------|

- 与动态控制配合：基础层→小动态，强调层→中动态，爆发/连击层→大动态

  交代环境+人物状态，一个匀速缓慢的普通动作

【升级】3-4s：
  景别=中景/近景，视角=平视/低角度，构图=越肩/对角线
  运镜=镜头推进或跟随，动态控制=中动态
  情绪或动作开始变化，景别切换

【高潮】2-3s：
  景别=特写/大特写，视角=低角度/高角度，构图=对角线/荷兰角
  运镜=镜头推进+旋转（叠加），动态控制=大动态
  关键动作或情绪爆发，速度变化（中动态蓄力→大动态释放）

【回落】3-5s：
  景别=中景/全景，视角=平视/高角度，构图=三分/对称
  运镜=镜头拉远或固定，动态控制=小动态→中动态
  余韵、转折或环境反应
```

**铁规则：**
- 时间块长度可以不等（如 5s+3s+2s+5s）
- 高潮点越短越有冲击力，禁止超过3秒
- 建立段可加长到6秒（大场景交代），但不可超过总时长50%
- 若该段无台词，回落段可用 **极远景+固定+小动态** 做纯画面留白

#### 10. 画面感自检清单（Agent写完提示词后逐条确认）

写完一段提示词后，必须检查以下5项。任一项不合格，重写该时间块：

1. **9维度齐全**：该时间块是否覆盖了主体、景别、视角、构图、运镜、动态控制、角色动作、环境动态、氛围？

4. **节奏清晰**：这一段的高潮点在第几秒？用什么景别+视角+构图+运镜+动态的组合到达的？
5. **张力存在**：如果去掉台词，这段画面本身还有情绪张力吗？

### TTS配音与口型同步（生产顺序铁律：先配音 → 再让画面跟着音频走）

> **核心原则**：AI生成视频时嘴部状态随机。先拿到精准音频，再用音频驱动口型同步，成功率远高于先画面后配音。
> 
> ❌ 错误：写台词 → 生成视频 → 贴配音 → 口型对不上 → 重抽
> ✅ 正确：写台词 → 生成配音 → 音频驱动口型 → 合成

---

#### 台词三层分类体系

每句台词必须标注类型，这是配音Agent和口型同步决策的基础。

| 类型 | 定义 | 配音 | 口型同步 | 画面要求 | 占比 |
|------|------|------|---------|---------|------|
| **核心对白** | 情绪高潮/Hook/关键剧情，必须让观众听到的台词 | 外部TTS全情绪配音 | **必须做** | 正面特写或近景，角色嘴部清晰 | ≤30% |
| **字幕台词** | 信息补充/过渡/次要对话，文字闪现即可 | 不配语音或只配轻语气音 | 不做 | 任意景别，字幕气泡闪现 | 50-60% |
| **旁白** | 画外音叙述，推进剧情 | 外部TTS配音 | 不做 | 空镜/背影/侧脸/极远景（角色嘴不动） | 10-20% |

**核心对白的技术限制：**
- 口型同步技术上限约 **≤6字/句**，超出需拆句
- 核心对白必须是 **正面特写或近景**，侧脸/背影/大角度无法做口型同步
- 连续核心对白不超过2句，否则口型疲劳

---

#### 口型同步决策树

```
这句台词需要口型同步吗？
  ├─ 核心对白 + 正面特写/近景 → 【必须做口型同步】
  ├─ 核心对白 + 侧面/角度大/背影 → 【换分镜为正面特写再做】
  ├─ 字幕台词 → 【字幕气泡闪现，不做口型】
  └─ 旁白 → 【切空镜/背影/侧脸，不做口型】
```

**分镜调整规则（为口型同步服务）：**
- 若剧本要求的核心对白当前分镜是背影/侧脸 → 视频Agent必须将该时间块分镜改为**正面特写或近景**
- 若核心对白句子太长（>6字）→ 台词Agent拆成连续短句（"跪下。"→"聋了？"→"跪下！"）
- 旁白时间块必须避免角色正面出镜，用空镜/背影/侧脸/环境特写替代

---

#### TTS工具链推荐

| 工具 | 优势 | 适用场景 | 情绪标签写法 |
|------|------|---------|-------------|
| **海螺AI** | 中文情绪还原好，抑扬顿挫自然 | 玄幻国漫首选，旁白和核心对白均可 | `[愤怒]` `[冷漠]` `[嘲讽]` `[绝望]` |
| **Fish Audio** | 音色克隆精度高，支持角色专属声线 | 需要固定角色声线时使用 | 先上传角色参考音频，生成专属Voice ID |
| **ElevenLabs** | 英文最强，多语言混合好 | 双语项目或有英文台词时 | `style="whispered"` `style="shouting"` |

**配音标注模板（每句核心对白/旁白必须附带）：**

```

角色：墨渊
TTS工具：海螺AI
情绪标签：[冷漠·轻蔑]

停顿：句首停顿0.3s，句尾停顿0.5s

```

#### 音频驱动口型同步工具

| 工具 | 能力 | 输入 | 输出 | 与即梦配合方式 |
|------|------|------|------|---------------|
| **可灵（Kling）** | 对口型效果较好，支持中文 | 视频+音频 | 口型同步后的视频 | 即梦生成无口型视频 → 可灵对口型 → 合成 |
| **即梦自带** | 支持，但效果不稳定 | 视频+音频 | 口型同步后的视频 | 直接在即梦网页端上传视频+音频 |
| **HeyGen** | 数字人口型极精准 | 视频+音频 | 口型同步后的视频 | 适合近景特写，成本较高 |

**推荐流程（性价比最优）：**
1. 即梦生成 **无口型视频**（提示词中不加任何口型/说话相关描述，避免干扰）
2. 外部TTS生成 **精准配音音频**
3. 可灵/即梦对口型工具将音频贴到核心对白时间块
4. 字幕台词用剪映/CapCut加字幕气泡
5. 旁白直接叠加音频，画面为空镜/背影

---

#### 台词密度与音频时长换算

| 视频时长 | 时间块数 | 核心对白 | 字幕台词 | 旁白 | 总字数 |
|---------|---------|---------|---------|------|--------|
| 15秒 | 4-5块 | 1-2句，≤12字 | 3-6句，20-40字 | 0-1句，0-10字 | 25-60字 |
| 每集75秒 | 5段×15秒 | 5-8句 | 15-25句 | 3-5句 | 约400-600字 |

**音频时长计算公式：**
- 核心对白时长 = 字数 ÷ 阅读速度(4字/秒) + 1.0s缓冲（含口型同步余量）
- 字幕台词时长 = 字数 ÷ 阅读速度(4字/秒) + 0.5s缓冲
- 旁白时长 = 字数 ÷ 阅读速度(5字/秒)（旁白语速可稍快）

## 九、提交与重试规则（视频生成）

### 核心原则
> **提交后 5 分钟未成功即判定 timeout，自动重提，最多重试 2 次。**

`auto_submit.py` 内置了智能轮询与自动重试机制，无需人工干预：

| 判定条件 | 检测方式 | 触发结果 |
|---------|---------|---------|
| **timeout** | 提交后 5 分钟内未完成（success/fail） | 自动重提，最多 2 次 |
| **orphan_stuck** | 连续 3 分钟查询无 queue_info（服务器孤儿任务） | 自动重提，最多 2 次 |
| **success** | 生成完成 | 自动下载并重命名 |
| **fail** | 生成失败（内容审核/其他） | 记录错误日志，放弃重试 |

### 轮询参数
- **轮询间隔**：30 秒 / 次
- **timeout 阈值**：5 分钟（`max_minutes=5`）
- **orphan 阈值**：连续 6 次无 queue_info = 3 分钟
- **最大重试次数**：2 次

### 重试日志
超时或孤儿任务会自动记录到 `任务记录/stuck_tasks.log`：
```
[2026-04-25T10:55:51] TIMEOUT 第09集01段-金茧碎裂.mp4 submit_id=xxx retry=1/2
```

### 为什么 timeout 设为 5 分钟？
- Fast 通道 15 秒视频实际生成时间通常 3-8 分钟
- 队列排到 `queue_idx=0` 后若超过 5 分钟仍未出结果，大概率是服务端卡死
- 不无限等待，及时重提，避免脚本挂死几小时

## 十、常见失败快速查表

| 现象 | 根因（真相文件定位） | 修复 |
|------|-------------------|------|
| 台词重复空洞 | 跳过剧本直接写提示词，台词现编 | 回到剧本Agent→台词Agent流程，先写剧本再生台词 |
| 台词太少像默片 | 密度参考过时，Agent不敢写 | 每段80-120字旁白+独白，全程有声音推进，打斗段也用旁白解说 |
| 台词死板无节奏 | 旁白写成流水账，缺乏爽文语感 | 用短句叠加/排比递进/先抑后扬/反问碾压四种节奏模板 |
| 口型对白孤零零 | 口型前后无旁白铺垫，观众听不懂 | 口型前加旁白建立上下文（如旁白"他缓缓抬头"→口型"跪下。"） |
| 旁白说明书化 | 旁白写成状态说明（"他很愤怒"） | 改为文学叙述+节奏感（"三年了，这笔账，该算了。"） |
| 角色说出解释性台词 | 台词未从场景逻辑生长 | 删除意图宣告型台词（"我要清算"），改为动作性台词（"跪下。"） |
| 续集开场变成上集场景 | timeline_truth未更新，跨场景接了@视频 | 更新timeline，删@视频，改全能参考 |
| 角色变脸 | book_rules角色锚点未注入 | 每段加@图片[N]，核对外形关键词 |
| 打斗模糊 | R08门禁未通过，全段单一速度无层次 | 基础层用"匀速缓慢"，高潮用"极慢蓄力→瞬间释放"或顿点式连击拆解 |
| 双角色穿模 | R09门禁未通过 | 加"两人相距约Xm，中间有明确空白距离" |
| 口型失败 | R10门禁未通过 | 单句拆到≤6字；用连续短句堆叠（"跪下。"→"聋了？"→"跪下！"） |
| 口型对不上 | 先画面后配音，AI嘴部随机 | 改为先TTS配音→音频驱动口型同步（可灵/即梦对口型） |
| 核心对白无正面镜头 | 分镜给的是背影/侧脸，无法做口型 | 核心对白时间块必须改为正面特写或近景 |
| 旁白时角色嘴在动 | 旁白时间块出现角色正面出镜 | 旁白时间块改空镜/背影/侧脸/环境特写 |
| 核心对白太多 | 超过总台词30%，口型疲劳 | 删减核心对白，改为字幕台词或旁白 |
| 生产顺序错误 | 视频在配音前生成 | 必须先TTS配音再生成画面，音频是驱动画面的锚点 |
| 配音情绪不对 | TTS未标注情绪标签 | 按配音标注模板写：情绪标签+语速+停顿+音频文件 |
| 三视图与立绘不符 | image2image一致性不足 | 多跑几次；退回网页端角色特征模式 |
| 转场双重溶解 | R14检查未运行 | 拼接两侧改硬切 |
| 首帧不对 | 未从视频截图 | 用ffmpeg从MP4提取 |
| 时间线倒退 | timeline_truth.json未填写 | 填写后运行gate_timeline_check |
| 画面催眠无张力 | 全段只有小动态，无大动态 | 按分镜规范加入大动态，高潮点用中动态蓄力→大动态释放 |
| 镜头语言单调 | 无景别切换或景别标注缺失 | 每15秒段必须≥2种景别（极远景/全景/中景/近景/特写/大特写） |
| 运镜写法空泛 | 写成"镜头缓缓移动"等非官方词 | 必须从官方15种基础运镜中选，高潮可用叠加运镜 |
| 节奏机械死板 | 时间块全为3秒均分，无高潮设计 | 按四节拍非均分划分（如5+3+2+5），明确标注Peak所在时间块 |
| 画面无法想象 | 缺少9维度信息 | 每个时间块必须覆盖主体+景别+视角+构图+运镜+动态+动作+环境+氛围 |
| 视角缺失 | 未标注视角导致情绪随机 | 必须标注低角度/高角度/平视/鸟瞰 |
| 构图缺失 | 无构图导致画面松散 | 必须标注对角线/三分/对称/越肩/荷兰角 |
| 环境静态 | 背景写成"废墟"无动态 | 必须写环境动态如"余烬飘落""气流卷起落叶" |
| 氛围突变 | 同一15秒内氛围无过渡突变 | 氛围必须一致或渐变，与book_rules色温对齐 |
| 生成失败content policy | 提示词含审核敏感词 | 按融图审核规避表替换：血液→暗红液体，杀→消散，爆炸→能量爆发，死亡→化为光点 |
| 主体描述孤立 | 元素无关系描写（"废墟。人物。剑。"） | 改为关系描写："人物手持剑站立于废墟中央，剑尖垂落地面" |
| 姿态缺失 | 含人物时间块未标注姿态 | 必须标注站立/跪地/侧身/挥剑等具体姿态 |
| 表情缺失 | 特写/大特写时间块未标注表情 | 必须标注冷漠/愤怒/痛苦/恐惧/嘲讽/悲伤 |
| 积分不足 | 未查余额直接提交 | 先`dreamina user_credit` |
| 任务被限流 | 视频并行提交过多 | 每次提交后sleep 2-3秒 |

### 并行提交规则

**并发限制按命令类型不同：**

| 命令 | 并发上限 | 说明 |
|------|---------|------|
| `text2image` | ✅ 多个 | 实测两个同时提交均成功，积分3/次 |
| `multimodal2video` | ❌ 1个 | 实测并发立即返回 ExceedConcurrencyLimit |

> ⚠️ **实测结论（2026-04-14）：multimodal2video 并发上限为 1，含排队中任务**
> - 飞书文档标注3个，但实测账号只允许同时存在1个任务（包括排队中的）
> - 提交第2个立即返回 `ExceedConcurrencyLimit (ret=1310)`
> - 并发提交时第一个上传成功、第二个上传同一张图会返回 `upload image: no file upload`——**根本原因是并行冲突，不是图片大小或格式问题**，压缩图片无法解决此问题
> - seedance2.0 和 seedance2.0fast 均受同一并发限制，模型版本不影响
> - **必须等上一个任务 success/fail 后再提交下一个**
> - 15秒 seedance2.0fast 实测消耗 **30积分**（非估算的50）
> - 队列深度可达42万+，queue_idx显示排队位置，越小越快出结果

- **最大并发数：1个**（含排队中，提交后必须等完成再提交下一个）
- 提交后轮询直到 success/fail，再提交下一段
- 串行提交模板：

```bash
BASE="C:/Users/DELL/Desktop/项目名"
LOG="$BASE/任务记录/错误日志.txt"

submit_video() {
  local label=$1; local prompt_file=$2; shift 2
  result=$(~/bin/dreamina.exe multimodal2video "$@" --prompt="$(cat $prompt_file)" 2>&1)
  exit_code=$?
  submit_id=$(echo "$result" | grep -o '"submit_id": "[^"]*"' | cut -d'"' -f4)
  status=$(echo "$result" | grep -o '"gen_status": "[^"]*"' | cut -d'"' -f4)

  if [ $exit_code -ne 0 ] || [ "$status" = "fail" ] || [ -z "$submit_id" ]; then
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] ❌ FAILED: $label" | tee -a "$LOG"
    echo "--- 报错内容 ---" >> "$LOG"
    echo "$result" >> "$LOG"
    echo "--- 提示词文件: $prompt_file ---" >> "$LOG"
    echo "--- 参考图: $@" >> "$LOG"
    echo "=================" >> "$LOG"
    return 1
  else
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] ✅ $label submit_id=$submit_id"
    # 轮询直到完成再提交下一段
    for i in $(seq 1 60); do
      sleep 30
      poll=$(~/bin/dreamina.exe query_result --submit_id="$submit_id" 2>&1)
      gen_status=$(echo "$poll" | grep -o '"gen_status": "[^"]*"' | cut -d'"' -f4)
      echo "[$(date '+%H:%M:%S')] $label 状态: $gen_status"
      if [ "$gen_status" = "success" ] || [ "$gen_status" = "fail" ]; then
        break
      fi
    done
  fi
}

# ⚠️ 串行提交（并发上限=1，禁止并行，每段等完成再提交下一段）
submit_video "EP01段01" "$BASE/提示词/视频/EP01_段01.txt" \
  --image "$BASE/素材/场景/场景-针叶山林-深夜.png" \
  --duration=15 --ratio=16:9 --model_version=seedance2.0

submit_video "EP01段02" "$BASE/提示词/视频/EP01_段02.txt" \
  --image "$BASE/素材/角色/角色-织-立绘正常.png" \
  --image "$BASE/素材/场景/场景-针叶山林-深夜.png" \
  --duration=15 --ratio=16:9 --model_version=seedance2.0

submit_video "EP01段03" "$BASE/提示词/视频/EP01_段03.txt" \
  --image "$BASE/素材/角色/角色-织-立绘正常.png" \
  --image "$BASE/素材/场景/场景-针叶山林-深夜.png" \
  --duration=15 --ratio=16:9 --model_version=seedance2.0

submit_video "EP01段04" "$BASE/提示词/视频/EP01_段04.txt" \
  --image "$BASE/素材/角色/角色-织-立绘正常.png" \
  --image "$BASE/素材/场景/场景-针叶山林-深夜.png" \
  --duration=15 --ratio=16:9 --model_version=seedance2.0

### 错误日志格式

所有视频生成失败自动写入 `任务记录/错误日志.txt`：

```
[2026-04-14 15:30:00] ❌ FAILED: EP01段02
--- 报错内容 ---
{"gen_status": "fail", "fail_reason": "content policy violation"}
--- 提示词文件: 提示词/视频/EP01_段02.txt ---
--- 参考图: --image 素材/角色/角色-织-立绘正常.png --image 素材/场景/场景-针叶山林-深夜.png ---
=================
```

**根据报错类型的修复方向：**

| 报错关键词 | 原因 | 修复 |
|-----------|------|------|
| `content policy` | 提示词触发审核 | 按融图审核规避表替换敏感词：血液→暗红液体，杀/死→消散/崩解，爆炸→能量爆发，恐怖→阴森 |
| `AigcComplianceConfirmationRequired` | 首次使用需网页端授权 | 去即梦网页完成授权确认 |
| `credit` / `insufficient` | 积分不足 | `dreamina user_credit` 查余额 |
| `EOF` / `timeout` | 网络超时 | 重新提交，加长 `--poll` |
| `ExceedConcurrencyLimit` | 并发超限（实测上限=1） | 改为串行，sleep 10s间隔 |
| `upload phase, no file upload` | 多进程同时上传图片冲突 | 改为串行，不并发上传 |
| `querying`（长时间） | 队列拥堵 | 检查是否误传了 `--model_version`（text2image禁止传） |
| `fail_reason` 为空 | 服务端未知错误 | 原样重试一次 |

### 模型通道参数（`--model_version`）

| 命令 | 可用值 |
|------|-------|
| `text2image` | ⚠️ 禁止传，传了进拥堵队列 |
| `image2image` | 4.0, 4.1, 4.5, 4.6, 5.0 |
| `image2video` | 3.0, 3.0fast, 3.0pro, 3.5pro, seedance2.0系列 |
| `multimodal2video` | seedance2.0, seedance2.0fast, seedance2.0_vip, seedance2.0fast_vip |
| `frames2video` | 3.0, 3.5pro, seedance2.0系列 |
| `text2video` | seedance2.0系列（默认seedance2.0fast） |
| `multiframe2video` | ❌ 不支持，ratio自动推断 |

**正片默认用 `seedance2.0fast`。旗舰 `seedance2.0` 出片极慢，仅特殊场景使用。**
